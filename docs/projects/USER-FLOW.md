# Fluxo do usuário

## Jornada principal

1. O usuário acessa a interface principal.
2. Escolhe entre colar o texto do email ou enviar um arquivo.
3. Envia a entrada para análise.
4. Aguarda o processamento.
5. Visualiza:
   - categoria do email
   - nível de confiança
   - justificativa resumida
   - resposta sugerida

## Objetivo de UX

O fluxo deve ser simples o suficiente para que uma pessoa entenda o valor da ferramenta em poucos segundos, sem depender de instruções externas.

## Estados que a interface deve cobrir

- estado inicial com orientação clara
- estado de carregamento
- estado de erro amigável
- estado de sucesso com resultado completo

## Elementos que devem ajudar a demo

- drag and drop ou input claro para upload
- exemplos prontos para teste
- destaque visual para a categoria classificada
- ação rápida para copiar a resposta sugerida
- feedback visual consistente em desktop e mobile
